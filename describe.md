# 项目核心：一个“智能 DnD 角色生成与背景故事助手”

## 你要做的不是一个普通车卡器，而是一个：

> **能根据职业 + 种族 + 阵营 + 玩家偏好，自动推荐世界观元素，并生成合理背景故事的智能助手。**

它包含三个核心模块：



# ① **经典 PDF 卡面复原（UI + 数据结构）**

这是你项目的“可见部分”，也是最容易让人惊艳的部分。

你可以做：

- 完整复刻 5e 角色卡的布局（属性、技能、装备、法术、背景）
- 支持自动填充（根据职业/种族自动算熟练、技能、生命值）
- 支持导出为 PDF（可选）
- 支持保存/加载角色

**亮点：**

- 展示你对前端/后端的掌握
- 展示你对 DnD 规则的理解
- 让你的项目“看起来很专业”



# ② **世界观推荐系统（RAG 或结构化知识库）**

玩家输入：

- 种族（如 Tiefling）
- 职业（如 Rogue）
- 阵营（如 CG）
- 背景（如 Urchin）
- 玩家偏好（如“想要悲剧感”“想要政治阴谋”）

系统自动推荐：

- 适合的城市（如 Waterdeep、Baldur’s Gate）
- 适合的组织（如 Harpers、Zhentarim）
- 适合的事件（如“被贵族收养”“卷入帮派冲突”）
- 适合的 NPC 关系（如“欠债的盗贼头目”）

你可以用：

- **结构化 JSON 数据（你自己整理）**
- **轻量 RAG（把世界观 wiki 文本切块）**
- **embedding + 相似度检索**

**亮点：**

- 展示你能把 LLM 和结构化数据结合
- 展示你能做一个“推荐系统”
- 展示你能做数据建模



# ③ **智能背景故事生成器（LLM）**

这是你项目的“AI 核心”。

输入：

- 职业、种族、阵营
- 推荐的城市/组织/事件
- 玩家偏好（如“想要轻松搞笑”）

输出：

- 一个 3–5 段的背景故事
- 包含合理的世界观细节
- 包含可用于 DM 的 plot hook
- 包含可用于玩家的动机与性格

你可以做：

- Prompt engineering（最简单）
- 加一点 RAG（可选）
- 让模型引用你推荐的世界观元素（强烈建议）

**亮点：**

- 展示你能做“AI 产品”，不是“AI 技术 demo”
- 展示你能做 prompt 设计
- 展示你能做“可控生成”

# 整个项目的大框架（你可以随时回来看）

你的项目分成 **5 个阶段**，按顺序做就不会乱：



## **阶段 1：选技术栈 + 搭环境（1–2 天）**

目标：让你能写代码、跑服务器、跑前端。

你需要决定：

- **前端**：React（最常用、资料最多）
- **后端**：Python + FastAPI（简单、轻量、适合 AI）
- **数据库**：不用数据库，先用 JSON 文件即可
- **AI**：OpenAI / DeepSeek / Qwen（后面再接）

阶段成果：

- 你能运行一个“Hello World”后端
- 你能运行一个“Hello World”前端
- 你能让前端访问后端 API



## **阶段 2：定义数据结构（1–2 天）**

## **阶段 3：后端基础逻辑（3–5 天）**

目标：让后端能：

- 创建角色
- 保存角色
- 加载角色
- 自动计算属性（熟练、技能等）
- 根据职业/种族做简单推荐（不用 AI）

成果：

- 一个可用的后端 API



## **阶段 4：前端 UI（5–7 天）**

目标：复刻经典 PDF 卡面。

成果：

- 一个可交互的角色卡界面
- 能输入/修改角色信息
- 能展示推荐结果



## **阶段 5：AI 模块（3–5 天）**

目标：

- 世界观推荐（embedding + 检索）
- 背景故事生成（prompt）
- 整合到前端

成果：

- 一个真正“智能”的 DnD 角色生成器
